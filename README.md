## Ce que j'ai compris des principes SOLID

- SRP: une classe ou un module doit avoir une seule responsabilite et donc une seule raison de changer.
- OCP: le code doit etre ouvert a l'extension, mais ferme a la modification; on ajoute du comportement via l'abstraction et le polymorphisme.
- LSP: un sous-type doit pouvoir remplacer son type de base sans casser le comportement attendu.
- ISP: il vaut mieux plusieurs petites interfaces ciblees qu'une grosse interface generale.
- DIP: les modules de haut niveau doivent dependre d'abstractions, pas de details concrets.
